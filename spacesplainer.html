Spacesplainer README

Spacesplainer is a space-to-LLM translator studio. It is for people who can arrange what they mean visually but do not want to explain every spatial relationship in prose. Build a GRID or DEPTH scene, copy the encoded packet, paste it into an LLM, and ask it to decode and use the layout.

What the output is

Outputs use DECODE_BASE64_JSON:. The encoded JSON contains the tool name, mode, interpretation rules, and spatial data. It is not an image file, not a CAD model, and not true 3D. It is structured layout guidance for an LLM.

GRID mode

GRID is the upgraded Pixel 2 Prompt idea. Use it for flat maps, tile layouts, icons, website sections, UI blocks, image composition regions, and negative space. Paint cells with colors, then use context labels to say what each color means.

  • View: tells the LLM how to interpret the grid. The default is screen/layout composition: like looking at a TV, website, poster, UI, or diagram, not looking down at a map.
  • Orientation: says whether top means north/east/south/west or has no direction.
  • Scale: says whether cells are exact tiles, rough placement, or abstract proportions.
  • Context: turns colors into meanings, like green = hero image or blue = article body.

DEPTH mode

DEPTH is for labeled objects that overlap, stack, or sit in front of/behind each other. Add an object, place it on the fine grid, and assign it a layer.

  • Layer: a depth plane, not a single slot. Many objects can share one layer.
  • Layer 1: closest/front. Higher layer numbers are farther back.
  • Focus Layer: dims other layers so crowded scenes are easier to edit.
  • Projection: tells the LLM how to read the scene. Stage depth means lower on the canvas is closer to the viewer, like theater seats or stage blocking.
  • Shape + Size: visual footprint hints. They help the LLM estimate how much objects overlap or cover each other.

Projection glossary

  • Flat: screen positions only, like looking at a flat screen or poster. No top-down map direction is implied.
  • Stage Depth: lower y positions are closer, higher y positions are farther back, while layer still controls overlap priority.
  • Overhead: top-down map or floor plan.
  • Front: looking straight at the scene.
  • Abstract: relationships matter, but no real physical view is implied.

How to use with an LLM

Copy a packet and paste it into your coding or image-prompting chat. Say something like: “Decode this base64 JSON Spacesplainer packet and use it as the spatial reference for the layout.”

Privacy & local data

Spacesplainer is intended to run in your browser. It does not require a server, account, analytics, microphone, camera, or file upload. Skin preference and the first-load overview setting are stored in browser localStorage. Copied packets only leave your device if you paste them somewhere else.

Tips

  • Use GRID when regions, cells, adjacency, and negative space matter.
  • Use DEPTH when front/back overlap matters.
  • Keep labels plain: player, wall, door, logo, sidebar, moon.
  • Use Focus Layer when objects are hiding each other.
  • Do not overbuild the scene. The packet is a translation aid, not a full design file.
