StorySplainer README

StorySplainer is a narrative context engine for LLMs. It does not write the story by itself. It builds a clean, encoded context handoff so an LLM can write or revise a scene with the world, nodes, relationships, and instructions already understood.

Core Idea

You can write a simple instruction like “the hero reaches the hill and hesitates.” StorySplainer adds the structured context: who the hero is, what the hill means, what object matters, which relationships are active, and what world tone should guide the scene.

World

The WORLD area is global context. It is pulled into every generation. Use the title and description for setting, tone, rules, constraints, genre, history, and background. The description can be longer than a tagline; give the LLM useful context when it matters.

World Controls

  • Import replaces the current world with pasted StorySplainer world JSON.
  • Export writes the current world, nodes, relationships, and saved scenarios as JSON.
  • Create starts a new blank world after confirmation. Export first if you want to keep the current one.

StorySplainer uses one active world at a time. Worlds are portable through import and export instead of a built-in multi-world switcher.

Nodes

PERSONS, PLACES, and THINGS are reusable story pieces. Each node has four compact signals: Name, Archetype, Sentiment, and Special. For example: John / Knight / Valor / Mysterious Origin.

Relationships

Relationships are short stacked signals between nodes. They are not hard logic rules. They are narrative voltage: “secret son,” “bitter rival,” “birthday present,” “skinned his knee here.” In Includes Only mode, relationships matter when both related nodes are included.

Scenario

A scenario is the working layer: scene includes, context mode, and special instructions. It is separate from the world. Reset Scenario clears includes, clears special instructions, and returns context mode to Includes Only. It does not clear the generated context code.

Context Mode

  • Includes Only sends selected nodes and relationships between those selected nodes.
  • Full World sends all nodes and all relationships for broader context.

LLM Context Code

The generated code is a DECODE_BASE64_JSON packet. Paste it into an LLM and ask it to decode and use it as the context reference. It is not a finished story. It is the structured handoff that helps the LLM write the story.

Storyboard

The STORYBOARD controls add optional visual output instructions to the scene packet. Choose 0–9 frames, select Photo Real, Illustration, or Pencil Sketch, and optionally use Storyboard Only when you want image prompts instead of narrative prose.

Bank

The BANK stores reusable scenario setups. Selecting a row previews the scenario; it does not load it. Use the explicit Load button when you want to apply it.

Privacy & Local Storage

StorySplainer stores data in browser localStorage. It does not upload your world, nodes, scenarios, or imports anywhere by itself. Imports are local JSON data processed in your browser. Do not put private secrets into content you plan to paste into an LLM.

StorySplainer

StorySplainer is a story-context-to-LLM translator. Build a world, define reusable nodes, assemble a scenario, and generate an encoded context packet.

SCENE builds the current setup. NODES stores people, places, things, and relationships. BANK saves reusable scenarios. README explains the system.

The point is simple: write what happens once, then let structured context carry the weight.
